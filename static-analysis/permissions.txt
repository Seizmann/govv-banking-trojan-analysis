## STAGE 1 DECLARED PERMISSIONS (3 external)

CRITICAL:
  android.permission.BIND_VPN_SERVICE
  → Allows creation of a VPN tunnel
  → Used to intercept 100% of device network traffic
  → addRoute("0.0.0.0", 0) confirmed in bytecode
  → This is the most dangerous permission in this APK

HIGH:
  android.permission.REQUEST_INSTALL_PACKAGES
  → Allows installing APK files without user prompt (after first grant)
  → Used to silently install b.apk (Stage 3 banking trojan)
  → Combined with InstallResultReceiver to monitor install

MEDIUM:
  android.permission.POST_NOTIFICATIONS
  → Allows posting notifications
  → Used for deceptive "Installing..." notifications

INTERNAL (self-defined):
  com.mmv.jnkh.rql7.tc8szt0d.DYNAMIC_RECEIVER_NOT_EXPORTED_PERMISSION
  → Custom permission for internal dynamic broadcast receivers
  → Limits external components from accessing internal broadcasts

## EVASION STRATEGY

The malware deliberately declares only 3 harmless-looking permissions
in Stage 1 (outer APK) to:
  1. Pass automated permission-based app scanners
  2. Avoid triggering "dangerous permission" warnings during install
  3. Reduce user suspicion during sideload

All truly dangerous permissions are requested at runtime by Stage 3
(b.apk), after the VPN is already active and the Accessibility Service
guide has tricked the user into granting permissions manually.

This is a deliberate, sophisticated evasion of permission-based detection.
